# 深圳报料数据驾驶舱 需求文档（PRD）

版本：0.3
日期：2026-07-04
赛题：8 小时单人全栈 AI 挑战赛 · 题目 1「报料数据驾驶舱 H5」
业务定位：深圳本地新闻媒体 · 报料线索运营驾驶舱
用户群体：新闻媒体运营人员

> 本文档为**纯产品需求文档**，只描述业务场景、用户、信息架构、功能与验收，不含技术实现方案（类型定义、目录结构、框架约束、构建配置等实现细节另见开发文档）。

## 1. 一句话定位

> 将分散的报料记录整合为**一张驾驶舱仪表盘**，运营人员**一眼掌握全局态势**：今日新增、处理进度、热点分布、趋势走向；并支持**多维度下钻与数据导出**。

这句话就是本产品的骨架。整份 PRD 都围绕它展开：

- **整合为一张** → 首页是一个整合的驾驶舱，不让运营人员到处找。
- **一眼掌握全局态势** → 首屏 30 秒内看懂当前状况，全是概览，不做重操作。
- **今日新增 / 处理进度 / 热点分布 / 趋势走向** → 驾驶舱首页的四类核心信息。
- **多维度下钻** → 从概览点进「工作台」看具体、做操作。
- **数据导出** → 把当前结果拉出来汇报。

## 2. 产品原则

1. **先判断，再解释，最后下钻。** 首页给判断（现在什么情况、该先看什么），工作台给操作（逐条处理、流转、导出）。
2. **观察与操作分离。** 只「看」的内容留在驾驶舱首页；需要「动手」的内容（分诊、状态流转、筛选导出）放进工作台。
3. **深圳本地真实感。** 区划、街道、来源平台、报道题材都用深圳真实素材抽象，不使用其他城市数据，不做随机假数据。
4. **不做装饰性大屏。** 做新闻编辑室真正会用的线索分诊台，信息密度优先，不堆营销式 Hero。

## 3. 用户与使用场景

| 用户角色 | 首要问题 | 需要的能力 |
| --- | --- | --- |
| 新闻运营人员（主用户） | 今天来了多少料、有没有积压、先分诊哪些 | 看驾驶舱态势、进线索明细「今日待办」逐条分诊 |
| 编辑室负责人 | 哪类线索、哪个区、哪个平台正在升温 | 看热点分布、趋势走向 |
| 记者 / 编辑 | 具体哪条线索值得跟进、处置轨迹是什么 | 在明细台筛选、排序、展开详情 |
| 管理人员 | 处理效率和采用转化是否健康 | 看处理进度、完成率、采用率，导出报表 |
| 评委 | 是否完成题目、是否专业、数据是否真实 | 30 秒看懂层次、交互、数据来源与移动端效果 |

**核心日常动线（运营人员）：** 打开驾驶舱看今日态势 → 发现待审积压 → 进「线索明细 · 今日待办」逐条分诊指派 → 到「处理流转」推进状态 → 需要汇报时在「线索明细」筛选并导出。

## 4. 信息架构

系统由**两个一级导航**组成：一个驾驶舱首页 + 一个工作台目录（下挂 3 个模块）。

```
【1】驾驶舱 · 首页
      一眼掌握全局态势，全是概览，不做重操作
      ├ 态势总览      今天该先看什么（可点击联动到工作台）
      ├ 今日新增 KPI  今日报料 / 平均处理时长 / 处理完成率 / 采用率 + sparkline
      ├ 处理进度概览  待审核 / 跟进中 / 已采用 / 不予采用 的流转
      ├ 热点分布      全国报料热力分布图（点击广东下钻深圳区划）+ 类型分布 + 来源渠道占比
      ├ 趋势走向      报料量 + 处理完成率双轴，日 / 周 / 月粒度
      └ 待办预览      Top N 高优先级线索（「查看全部」进工作台）

【2】工作台 · 目录（承载操作与下钻）
      ├ 线索明细    全量筛选 / 排序 / 行展开 / 逐条分诊指派 / CSV 导出
      │             含「今日待办」快捷筛选；接收首页图表下钻
      └ 处理流转    按状态流转的看板：待审核 → 跟进中 → 已采用 / 不予采用
```

**观察 vs 操作的边界：**

- 驾驶舱首页任何板块**只回答「现在什么情况」**，需要深入的一律引导进工作台。
- 首页图表点击「下钻」= 跳到工作台「线索明细」并自动带上对应筛选。
- 热点分布、趋势走向不单独成页，它们的下钻就是联动到线索明细。

**首屏顺序（桌面 / H5 一致，纵向堆叠）：** 顶栏工具区 → 态势总览 → 今日新增 KPI → 处理进度 + 热点分布 → 趋势走向 → 待办预览。

**移动端（375px）：** 工具栏折叠为两行；KPI 单列或两列；概览图单列；表格横向滚动、核心列固定。

## 5. 功能需求

### 5.1 全局工具区（贯穿所有页面）

| 功能 | 说明 | 验收要求 |
| --- | --- | --- |
| 时间范围快捷选择 | 今天 / 本周 / 本月 / 自定义 | 切换后驾驶舱与工作台数据联动 |
| 自定义日期范围 | 选择起止日期 | 无数据时显示空状态，不报错 |
| 刷新 | 重新生成 / 拉取当前数据 | 按钮有 loading 态 |
| 最后更新时间 | 展示当前数据生成 / 刷新时间 | 常驻可见 |
| 主题切换 | 明 / 暗主题 | 全站生效 |
| 数据导出入口 | 导出当前筛选结果 | 见 5.6 |

### 5.2 驾驶舱首页

首页只做概览，回答「现在整体什么情况、该先看什么」。

**① 态势总览**

把图表数据转成编辑室可行动的判断。有数据时展示 3 条，每条包含「发现了什么」+「建议做什么」，可点击联动。

| 摘要类型 | 触发规则 | 点击动作 |
| --- | --- | --- |
| 待审压力 | 待审核占比过高，或高优先级线索偏多 | 进工作台并筛待审 / 高优先级 |
| 区域高发 | 某区线索数明显居首 | 进明细并筛该区 |
| 类型升温 | 某类型较上一周期明显增长 | 进明细并筛该类型 |
| 渠道异常 | 某来源 / 渠道当天占比异常升高 | 进明细并筛该渠道 |
| 响应变慢 | 平均处理时长较上一周期上升 | 进明细按响应时长排序 |
| 突发事件 | 存在未完成的突发事件线索 | 进工作台筛突发 + 高优先级 |

无异常时显示「当前态势平稳，暂无明显异常」，不留空。

**② 今日新增 KPI**

四张核心指标卡，每张含：指标名、主数值、环比箭头、环比文案、sparkline 迷你趋势、口径 tooltip。

| 指标 | 计算口径 |
| --- | --- |
| 今日报料数 | 今日 00:00 至当前的记录数 |
| 平均处理时长 | 非待审核线索从提交到首次响应的平均分钟数 |
| 处理完成率 | （已采用 + 不予采用）/ 当前范围全部线索 |
| 线索采用率 | 已采用 /（已采用 + 不予采用） |

**③ 处理进度概览**

展示待审核 / 跟进中 / 已采用 / 不予采用 四个状态的分布与流转（横向堆叠条或小漏斗）。点击任一状态 → 下钻到线索明细并筛该状态。

**④ 热点分布**

| 可视化 | 形式 | 展示 | 交互 |
| --- | --- | --- | --- |
| 全国报料热力分布图 | 两级填色地图（choropleth） | 第一级：全国省级密度（mock 仅广东/深圳有数据，其余灰底占位）；第二级（点击广东下钻）：深圳 9 行政区 + 大鹏新区的线索密度、待审数、采用率，深汕特别合作区作为独立 chip 补充 | 全国级：点击广东下钻；深圳级：桌面 hover 显详情 / 移动端点选出详情条 + 点击下钻明细；卡片内显式「返回全国」按钮回退 |
| 线索类型分布 | 条形图 | 突发 / 民生 / 交通 / 环境 / 文体 / 消费 / 其他 | hover + 点击筛该类型 |
| 来源渠道占比 | 环形图 | 各深圳新闻平台 / 报料渠道占比 | hover + 点击筛该来源 |

热力分布图要求：两级地图共用同一张卡片，同卡片内切换而非并列展示；引入简化几何（示意轮廓，非精确测绘级别），用颜色深浅表达密度；全国级仅广东/深圳有 mock 数值、其余省份灰底不可点；深圳级高待审区域显示克制的 warning 描边；移动端优先，点选驱动交互（点击区块 → 高亮 + 底部详情条 + 下钻按钮），桌面端额外支持 hover 预览；从深圳级返回全国级用卡片内显式按钮，不依赖双击或点击空白区域。

**⑤ 趋势走向**

双轴折线：左轴报料量，右轴处理完成率 / 采用率。支持日 / 周 / 月粒度切换，受时间范围联动。tooltip 同时显示线索数、完成率、采用率。当前范围无数据时显示说明与重置入口。

**⑥ 待办预览**

首页底部展示 Top N 条高优先级线索（编号、标题、区域、优先级、状态），「查看全部」进入工作台。

### 5.3 工作台 / 线索明细

全量数据查询台，同时承担**逐条分诊**与**首页图表下钻**两个职责。既能像收件箱一样处理今天新进的料，也能像查询台一样多维筛选并导出。

**「今日待办」快捷视图：** 提供一键切到「今日新进 / 待审核」的快捷筛选，默认按优先级 + 时间排序，作为运营人员每天的第一入口；处理后可从待办视图弱化或移出。

**表格字段：** 编号、标题（截断 + tooltip）、来源平台、报料渠道、类型、区域（区 + 街道）、状态、优先级（+ 命中原因）、报料时间、处理时长（待审显示 `-`）、跟进人。

**筛选维度：** 状态、类型、来源平台、报料渠道、深圳区域、优先级、日期范围；含「今日待办」快捷筛选。

**查询能力：** 报料时间 / 处理时长 / 优先级排序、分页、行 hover 高亮、行展开、空状态 + 清空筛选入口。

**分诊能力：** 对单条线索定优先级、标类型、指派跟进人、推进状态（待审核 → 跟进中 → 采用 / 不采用）。

**行展开详情包含：** 线索描述、参考新闻题材来源、报料人脱敏信息、地点（区 / 街道 / 地点名）、处理轨迹（提交 → 分拨 → 审核 → 跟进 → 采用 / 不采用）、优先级命中原因。

### 5.4 工作台 / 处理流转

按处理状态组织的看板，看每条料卡在哪个环节、有没有积压。

| 功能 | 说明 |
| --- | --- |
| 状态分列 | 待审核 / 跟进中 / 已采用 / 不予采用 分列展示 |
| 积压感知 | 每列显示数量，超阈值列给出提醒 |
| 状态流转 | 支持将线索在状态间推进 |
| 卡片信息 | 展示编号、标题、区域、类型、优先级、停留时长 |

### 5.6 数据导出

导出**当前筛选后**的结果（不是全量固定数据）。

- 中文不乱码（UTF-8 with BOM），Excel 可正常打开。
- 文件名含日期时间，如 `深圳报料线索_YYYYMMDD_HHmm.csv`。
- 文件顶部写入导出时间、筛选条件、记录数、数据口径。
- 字段含编号、标题、来源平台、报料渠道、类型、区域、街道、状态、优先级、报料时间、处理时长、跟进人、描述。
- 0 条结果时禁用导出或给出明确提示。

### 5.7 图表联动与筛选反馈

| 交互 | 行为 |
| --- | --- |
| 点击态势总览 | 进工作台并设置对应筛选 / 排序 |
| 点击处理进度状态 | 下钻明细并筛该状态 |
| 点击全国地图上的广东 | 卡片内切换到深圳区划热力分布图 |
| 点击深圳区划热力分布图 | 下钻明细并筛该区 |
| 点击「返回全国」 | 卡片内切回全国热力分布图 |
| 点击类型 / 来源图表 | 下钻明细并筛该类型 / 来源 |
| 关闭单个 chip | 移除该筛选，不影响其他 |
| 清空全部 | 保留时间范围，清除其他筛选 |

筛选状态必须有可见反馈：顶栏下方显示 chips；选中的图表元素高亮、未选中降低透明度；表格标题显示「当前筛选 N 条 / 当前范围 M 条 / 全量 T 条」。

### 5.8 线索优先级预警

优先级由规则派生，辅助而非替代编辑判断，行展开需展示命中原因。

| 优先级 | 规则（示例口径） | 展示 |
| --- | --- | --- |
| 高 | 突发事件未完成；待审超时；处理时长过长；暴雨 / 火情 / 事故 / 内涝等应急标签 | 红色 Badge「需优先处理」 |
| 中 | 跟进超时；民生 / 环境类来自问政深圳 / 热线 / 报料小程序；同区同类当日高发 | 橙色 Badge「持续跟进」 |
| 低 | 已采用 / 不予采用，或无异常命中 | 灰色 Badge「常规」 |

## 6. 深圳本地化事实源

模拟数据不复制新闻全文，只从公开深圳新闻平台抽象题材、地点、渠道与处置关系。

### 6.1 区划

深圳下辖 9 个行政区 + 大鹏新区（功能区），另有深汕特别合作区。主驾驶舱区划使用：

福田、罗湖、盐田、南山、宝安、龙岗、龙华、坪山、光明（9 行政区）+ 大鹏新区。深汕特别合作区作为「跨域 / 应急」补充区域，可进入表格与趋势，不强制进首屏热区前 10。

- 地图主体展示 9 行政区 + 大鹏新区（全部同时可见，无需分页 / Top10 截断）；深汕特别合作区因地理飞地属性单独渲染为角落 chip。
- **不得出现天河、越秀、海珠、荔湾等广州或其他城市区划。**
- 每个区配真实街道样例（如福田—华强北 / 香蜜湖，龙岗—布吉 / 坂田，南山—蛇口 / 西丽 等）。

| 来源平台 | 用途 |
| --- | --- |
| 深圳新闻网 | 本地新闻与问政线索样本 |

| 读特客户端 | 交通、突发、民生深度报道样本 |
| 深圳特区报 | 应急、城市治理、政策类线索 |
| 读创 / 深圳商报 | 商圈、消费、营商环境类线索 |

报料渠道：报料小程序、新闻热线电话、微信公众号、客户端爆料、微博、短视频平台、现场投递。

## 7. 数据维度要求

以下为**业务维度要求**（字段的产品含义），不含数据结构定义。

**单条线索需承载：** 编号、标题、描述、类型、来源平台、报料渠道、状态、区域、街道 / 地点、脱敏报料人、跟进人、相关处置单位、报料时间、首次响应时间、处理时长、参考题材、来源链接、风险标签、处理轨迹。

**派生信息：** 优先级与命中原因、区域聚合统计、来源聚合统计、状态聚合统计、态势摘要、当前筛选计数。

**规模与分布：**

| 维度 | 要求 |
| --- | --- |
| 总量 | 覆盖约 180 天、600 条以上 |
| 区域覆盖 | 9 行政区 + 大鹏新区都有数据，深汕可有应急数据 |
| 类型覆盖 | 每类足量，突发 / 民生有明显波动 |

## 8. 验收标准

### 8.1 功能

- 驾驶舱首页可直接访问，30 秒内看懂当前态势。
- 工作台两个模块（线索明细 / 处理流转）均可访问，线索明细含「今日待办」快捷视图。
- KPI 四卡完整，含环比与 sparkline。
- 至少 5 类可视化：sparkline、环形图、条形图、全国 / 深圳两级热力分布图、双轴趋势（可加状态进度）。
- 今天 / 本周 / 本月 / 自定义 时间范围均可用；日 / 周 / 月粒度可切换。
- 图表 hover 有 tooltip；来源 / 类型 / 区划 / 状态点击可下钻筛选明细。
- 明细支持筛选、排序、分页、行展开；筛选 chips 可单独清除与全部清空。
- CSV 导出当前筛选结果，中文不乱码；空状态可恢复。

### 8.2 数据

- 总量 600 条以上，当前范围至少 30 条可分析。
- 9 行政区 + 大鹏新区都有数据；不出现广州或其他城市区划。
- 至少 8 个深圳来源平台 / 报料渠道维度。
- 每条有区、来源平台、类型、状态、时间、处理轨迹；至少 50% 带街道 / 地点。

### 8.3 响应式

- 375px 无文字重叠；表格横向滚动不撑破页面。
- 768px 图表不挤压。
- 1440px 首屏能看到 KPI、态势总览与至少两个概览图。
- 图表固定高度，加载与空状态不导致布局跳动。

## 9. 风险与处理

| 风险 | 处理 |
| --- | --- |
| 深圳素材抓取过度变成版权复制 | 只抽象题材 / 地点 / 类型，不复制长段正文 |
| 自定义日期实现拖慢 | 先做简版日期输入，不追求复杂日历 |
| 图表联动状态混乱 | active chips + 图表选中态 + 清空入口 |
| 简化地图被误当精确测绘 | 标题标注「示意地图」，几何为简化轮廓，展示区名 / 密度 / 待审 / 采用率 |
| 全国地图被误当真实统计覆盖全国 | 明确说明「仅广东 / 深圳有 mock 数据，其余省份灰底占位」，其余省份不可点击 |
| 移动端表格难用 | 核心列前置，外层横向滚动，展开详情纵向阅读 |
| 预警规则像黑盒 | 行展开与 tooltip 展示命中原因 |
