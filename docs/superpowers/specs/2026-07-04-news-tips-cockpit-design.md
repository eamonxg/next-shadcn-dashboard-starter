# 新闻报料线索驾驶舱 — 设计文档

> 8 小时单人全栈 AI 挑战赛 · 题目1「报料数据驾驶舱 H5」
> 日期：2026-07-04 ｜ 场景：广电媒体新闻线索运营

## 1. 背景与目标

公司（广播电影/广电媒体）的报料业务指观众、网友通过多渠道提供的新闻线索。本项目将分散的线索记录整合为一张驾驶舱式数据看板，供新闻运营/编辑室负责人一眼掌握：今日新增、审核跟进进度、热点类型分布、渠道来源、趋势走向，并支持多维筛选下钻与数据导出。

**成功标准（对齐评分表）：**

- 信息架构（30分）：KPI → 概览 → 趋势 → 明细四层数据故事线
- 图表品质（20分 + 加分5分）：≥4 种图表类型，全部可交互（tooltip/切换/联动）
- 交互体验（20分）：≥3 项可操作功能，实做 8 项，均有即时反馈
- 数据真实感（15分）：约 200 条模拟线索，KPI/图表由明细实时聚合，数字自洽
- UI 品质（15分）：双主题（默认深色大屏风，可切浅色），风格统一

## 2. 关键决策（已确认）

| 决策点 | 结论 |

| 技术路线 | 沿用 Next.js 16 + shadcn 模板，部署 Vercel 交云端链接，zip 附源码 |

## 3. 页面结构（多页驾驶舱系统）

路由：`/` 重定向到 `/dashboard/news-tips`。系统不局限于单页，按使用任务拆为总览、仪表盘和明细三个主页面：

| 页面 | 路由 | 职责 |
| --- | --- | --- |
| 总览驾驶舱 | `/dashboard/news-tips` | 首屏态势判断：KPI、态势摘要、关键概览、待处理线索预览 |
| 数据仪表盘 | `/dashboard/news-tips/analytics` | 图表分析：来源、类型、状态、深圳区划热区、趋势粒度切换 |
| 线索明细台 | `/dashboard/news-tips/records` | 表格重操作：筛选、排序、分页、展开详情、CSV 导出 |
| 设计规范 | `/dashboard/design` | 色板、字体、组件样式，可保留为交付说明页 |

总览驾驶舱自上而下：

1. **顶栏工具区**：页面标题、时间范围快捷选择（今天/本周/本月/自定义区间）、刷新按钮（带 loading）、明暗主题切换
2. **页面内导航**：总览 / 数据仪表盘 / 线索明细
3. **KPI 行**（4 卡）：今日新增线索、本周线索总量、平均响应时长、处理完成率；每卡含环比箭头（对上一周期）+ sparkline 迷你趋势
4. **态势摘要**：三条“发现了什么 + 建议做什么”
5. **关键概览**：来源平台占比、深圳区划热区 Top 区域、状态压力摘要
6. **待处理预览**：高优先级或待审核线索的轻量列表，跳转明细页继续处理

数据仪表盘自上而下：

1. 来源平台环形图
2. 线索类型条形图
3. 处理状态进度图
4. 深圳区划热区矩阵
5. 双轴趋势分析，支持日 / 周 / 月

线索明细台自上而下：

1. 筛选 chips 和当前数量口径
2. 表格工具栏：状态、类型、来源平台、渠道、区域、优先级、日期、排序、导出
3. TanStack Table：分页、排序、横向滚动、行展开
4. 行展开详情：描述、参考题材、脱敏报料人、地点、处理轨迹、优先级原因

图表类型合计 ≥4：sparkline（迷你面积）、环形图、条形图、双轴折线图，全部 Recharts、全部含 hover tooltip。

**加分项（仅主流程完成后做）**：更精细的深圳区划伪地图式热区矩阵；环形图点击扇区联动筛选明细页。

## 4. 数据模型

新建 feature 模块 `src/features/news-tips/`，遵循模板三件套：

```
src/features/news-tips/
  api/types.ts     # 类型契约
  api/service.ts   # mock 数据生成 + 查询/聚合函数
  api/queries.ts   # React Query queryOptions + key factory
  components/      # section-nav、KPI 卡、图表卡、明细表等
  utils/           # 聚合、CSV 导出等纯函数
```

**线索记录（NewsTipRecord）字段：**

| 字段 | 说明 |
| --- | --- |
| id | 线索编号，如 `BL-20260704-001` |
| title | 线索标题 |
| description | 线索详情描述（行展开显示） |
| category | 类型：突发事件 / 民生投诉 / 交通出行 / 文体娱乐 / 环境城建 / 其他 |
| channel | 来源：新闻客户端APP / 微信公众号 / 新闻热线电话 / 微博 / 短视频平台 / 现场投递 |
| status | 状态：待审核 → 跟进中 → 已采用 / 不予采用 |
| district | 区域（本地行政区） |
| reporter | 报料人（脱敏昵称） |
| assignee | 跟进记者/编辑 |
| createdAt | 报料时间 |
| responseMinutes | 响应时长（分钟，待审核为空） |
| timeline | 处理轨迹（提交→审核→跟进→结果，行展开显示） |

**Mock 策略**：确定性种子随机生成约 200 条，覆盖近 30 天，分布有意设计（如工作日多于周末、突发类占比波动），保证图表有可读的形态。KPI 与所有图表数据由明细数组在 service 层实时聚合，保证口径自洽。

**纯前端约束**：

- `api/service.ts` 是本地 deterministic mock service，不是后端服务，不调用数据库、Route Handler、Server Action、BFF、外部接口或 AI 总结接口。
- 所有筛选、聚合、图表、态势摘要、优先级和 CSV 导出均在本地 mock 数组上完成。
- 允许使用 Next.js Server Component 进行静态预渲染，但不得依赖 `headers()`、`cookies()`、请求上下文、环境密钥、Node 文件系统或运行时网络请求。
- 页面应可通过 Next.js `output: 'export'` 打成纯静态文件；静态构建时关闭 Sentry tunnel/rewrite，并启用 `images.unoptimized`。
- 与驾驶舱无关的模板示例 API Route 如果阻碍 static export，应在最终清理阶段移除或禁用。

## 5. 交互清单（8 项）

1. 全局时间范围切换（联动 KPI/图表/表格）
2. 趋势图粒度切换（日/周/月）
3. 表格多维筛选（状态/类型/来源/日期）
4. 表格列排序
5. 表格分页
6. 行点击展开线索详情与处理轨迹
7. CSV 导出当前筛选结果（Blob 真实下载）
8. 刷新（loading 反馈）+ 明暗主题切换

## 6. 模板改造

- 移除 Clerk 鉴权拦截（middleware 放行），根路径直达驾驶舱
- 导航精简：侧边栏仅保留驾驶舱与 `/design` 设计规范页；隐藏/移除 product、kanban、chat 等示例页入口
- 不新增 `/api/news-tips` 或任何后端接口；news-tips 只读本地 mock service
- `next.config.ts` 增加静态导出开关，例如 `NEXT_STATIC_EXPORT=true` 时使用 `output: 'export'`
- 复用模板的 PageContainer、Card、data-table、主题系统、Icons 注册表；遵循 CLAUDE.md 全部规范

## 7. 错误与边界处理

- 筛选无结果：表格空状态插画/文案 + 一键清空筛选
- 自定义日期区间无数据：图表空状态提示
- CSV 导出零行时按钮禁用
- 移动端：KPI 两列换行、图表纵向堆叠、表格横向滚动
